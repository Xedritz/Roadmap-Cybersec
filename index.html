Errores comunes — evítalos activamente
❌ Errores que destruyen el progreso
  • Tutorial hell: Ver videos sin practicar. Regla: por cada 30 min de teoría, 90 min de práctica.
  • Saltar fases: Querer hacer OSCP sin conocer Linux. Las fases tienen un orden por razón.
  • No documentar: Hacer labs sin tomar notas. Usas Obsidian o Notion desde el día 1.
  • Certificaciones sin práctica: Cert-dumping. Sin labs reales, las certs no sirven en entrevistas técnicas.
  • Aislamiento: No unirse a comunidades. Discord de TryHackMe, HTB, r/netsec son recursos infinitos.
  • Herramientas antes que conceptos: Usar Metasploit sin entender qué hace el exploit debajo.
✅ Aceleradores de aprendizaje
  • Active recall: Después de cada sesión, cierra todo y escribe 5 cosas que aprendiste sin mirar.
  • Enseña lo que aprendes: Escribir un blog post te obliga a entender de verdad.
  • Build > watch: Construir tu propio scanner de 100 líneas > ver 10 videos sobre Nmap.
  • Falla en público: CTFs, bug bounty sin experiencia. El fracaso controlado acelera el aprendizaje.
  • Spaced repetition: Usar Anki para conceptos teóricos (CIDRs, event IDs, CVSSv3).
  • Hackear tu propio sistema: Los mejores defensores piensan como atacantes. Siempre.
Cómo conseguir tu primer trabajo en ciberseguridad
🎯 Posiciones de entrada realistas (Mes 10–12)
  • SOC Analyst Tier 1: Monitoreo de alertas SIEM, triage de incidentes. Security+ + TryHackMe SOC Path es suficiente.
  • Cloud Security Analyst: Gestión de postura en Azure/AWS. AZ-500 es muy valorada aquí.
  • Junior Penetration Tester: Requiere eJPT mínimo + portafolio. Mejor con CEH o OSCP en progreso.
  • Security Engineer (DevSecOps): Empresas tech que buscan integrar seguridad en pipelines. GitHub Actions + Terraform + coding skills.
  • GRC/Compliance Analyst: Más fácil de entrar. ISO 27001, SOC 2. Valoran certificaciones.
💼 Estrategia de búsqueda activa
  • LinkedIn con keywords: "Cybersecurity", "Cloud Security", "SOC Analyst". Conectar con HR y managers directamente.
  • GitHub visible: Cada proyecto con README en inglés y español. Los reclutadores buscan GitHub.
  • Comunidades locales: OWASP chapter local, BSides conferences, meetups de seguridad.
  • Hackerrank / Leetcode: Algunas empresas tienen pruebas técnicas de código. Python básico suficiente.
  • Aplicar incluso sin 100% de requisitos: Si cumples 70%, aplica. Las JDs siempre son aspiracionales.
  • Bug bounty como experiencia: "Identifiqué y reporté [tipo] en programa de bug bounty" en el CV.
🚀 Plan de 5 años post-empleo
  • Año 1: Consolidar en primer rol. Aprender de colegas. OSCP si es posible.
  • Año 2: Especialización: Red Team Lead, Cloud Security Architect o DFIR Senior.
  • Año 3: CISSP (requiere 5 años experiencia). Senior roles, $80-120k USD.
  • Año 4-5: Principal/Lead Security Engineer, Security Architect, CISO junior. $120-180k+ USD.
  • Alternativa: Freelance consulting, bug bounty full-time, pentesting firm propio.
⚡ Las 5 reglas de oro
  • Práctica todos los días: 2h/día durante 12 meses > 20h/semana por 3 meses. La consistencia gana.
  • Lab propio es obligatorio: Sin hands-on en tu propio entorno, no hay progreso real.
  • Documenta TODO: Obsidian para notas, GitHub para proyectos, blog para publicar.
  • Busca mentores: LinkedIn outreach a profesionales seniors. El 20% responde. El 5% puede cambiar tu carrera.
  • Ética antes que técnica: Nunca en sistemas sin autorización. La reputación lo es todo en esta industria.

📚 Libros recomendados
📗
The Linux Command Line — William Shotts GRATIS online
Fase 0–1 · La biblia del CLI Linux. Disponible gratis: linuxcommand.org/tlcl.php
📕
Penetration Testing — Georgia Weidman ~$35 USD
Fase 3 · El libro de pentesting más recomendado para principiantes. Cubre metodología, Metasploit, shellcode, AD, mobile. nostarch.com/pentesting
📙
The Hacker Playbook 3 — Peter Kim ~$30 USD
Fase 3–4 · Técnicas avanzadas de Red Team: AD attacks, evasión, C2. Muy práctico y actualizado. Amazon
📘
Web Application Hacker's Handbook — Stuttard & Pinto ~$40 USD / PDF gratis (buscar)
Fase 3 · La referencia definitiva de web hacking. Aunque es del 2011, los conceptos de SQLi, XSS, auth bypass siguen siendo 100% válidos. Wiley
📓
OWASP Testing Guide v4.2 GRATIS
Fase 3 · Guía oficial de testing de aplicaciones web de OWASP. 500+ páginas de metodología y técnicas. owasp.org/testing-guide

🛠 Herramientas online esenciales (siempre abiertas)
Shodan — Buscador de dispositivos
Gratis + Pro
Motor de búsqueda para dispositivos conectados a internet. Útil para OSINT, verificar exposición de servicios, encontrar configuraciones vulnerables. Plan básico gratis con registro.
→ shodan.io
CyberChef — Swiss army knife digital
GRATIS
Herramienta web para operaciones de datos: encoding/decoding (base64, hex, URL), cifrado, hashing, análisis forense. Imprescindible en CTFs y análisis de malware. Funciona offline también.
→ gchq.github.io/CyberChef
MXToolbox — Diagnóstico email/DNS
GRATIS
Suite de herramientas DNS, email y red: lookup MX, SPF/DKIM/DMARC verification, blacklist check, header analysis de emails. Esencial para análisis de phishing y reconocimiento.
→ mxtoolbox.com
CVSS Calculator (NVD)
GRATIS
Calculadora oficial de CVSS v3.1 del NIST. Úsala para asignar scores a vulnerabilidades en tus reportes de pentest. También busca CVEs con sus scores en el NVD (National Vulnerability Database).
→ nvd.nist.gov/cvss-calculator
Have I Been Pwned
GRATIS
Verifica si emails o contraseñas aparecen en brechas de datos conocidas. API gratuita para integrar en scripts Python. Útil para demostrar riesgo en reportes y para reconocimiento OSINT.
→ haveibeenpwned.com
Obsidian — Notas técnicas
GRATIS
La mejor herramienta para tomar notas técnicas. Markdown local, graph view, templates, plugins para CTF notes. Muchos profesionales de ciberseguridad documentan todo su conocimiento aquí.
→ obsidian.md
